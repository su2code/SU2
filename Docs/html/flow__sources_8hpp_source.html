309 /*!
310  * \class CSourceVorticityConfinement
311  * \brief Class for a source term due to vorticity confinement.
312  * \ingroup SourceDiscr
313  * \brief Vorticity Confinement (VC) technique to counter the numerical
314  * diffusion offered by the numerical scheme
315  * \author: Y Chandukrishna, Josy P Pullockara, T N Venkatesh.
316  * Computational and Theoretical Fluid Dynamics division,
317  * CSIR-National Aerospace Laboratories (NAL), Bangalore.
318  * Academy of Scientific and Innovative Research (AcSIR), Ghaziabad.
319  * First release date : 23 December 2022
320  * modified on:
321  *
322  * VC technique introduces an additional term to the N-S equations that
323  * counters the numerical difusion offerd by the numerical schemes. The
324  * additional term is introduced as a source term. VC technique requires an
325  * input confinement parameter that controls the magnitude of the source
326  * term. A suitable Confinement parameter is problem, scheme and grid
327  * dependant.
328  *
329  * Required changes in config file -
330  * VORTICITY_CONFINEMENT = YES
331  * CONFINEMENT_PARAMETER = <0.0(default)>
332  * Currently have support for Viscous, Inviscid cases for both 2D and 3D
333  * problems. Can be used along with other source terms also.
334  *
335  * The current implementation follows,
336  * R. Loehner and C. Yang, Vorticity confinement on unstructured grids, 2002.
337  * N. Butsuntorn and A. Jameson, Time Spectral Method for Rotorcraft Flow, 2008.
338  */
339 class CSourceVorticityConfinement final : public CSourceBase_Flow {
340 public:
341  /*!
342  * \brief Constructor of the class.
343  * \param[in] val_nDim - Number of dimensions of the problem.
344  * \param[in] val_nVar - Number of variables of the problem.
345  * \param[in] config - Definition of the particular problem.
346  */
347  CSourceVorticityConfinement(unsigned short val_nDim, unsigned short val_nVar, const CConfig* config);
